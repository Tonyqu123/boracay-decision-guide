# 长滩岛住哪里？

回答 6 个问题，告诉你长滩岛该住哪个区域，再给几个酒店方向。

这个项目现在只做一个很小的商业验证闭环：

```text
首页 -> 6 个问题 -> 免费基础结果 -> 模拟解锁 -> 完整住宿决策卡
```

不做完整旅行规划，不做导购，不做复杂搜索，也暂时不接真实支付。

---

## 当前 MVP

用户只经历三步：

1. 在首页点击开始测试。
2. 回答 6 个固定选项问题。
3. 得到免费基础结果：适合区域、真实需求、备选、不建议。
4. 进入模拟解锁页。
5. 查看完整住宿决策卡：判断理由、酒店检查点、酒店候选、PDF / 图片下载。

第一版判断区域只有：

- Station 1
- Station 2
- Station 3
- Bulabog
- Diniwid
- Yapak / 北部度假村区

---

## 运行方式

这是一个纯静态页面。

建议用静态服务器打开：

```bash
python3 -m http.server 8089
```

然后访问：

```text
http://localhost:8089/
```

页面路径：

```text
/
/quiz/
/result/?id=...
/pay/?id=...
/success/?id=...
```

---

## 实现方式

- `index.html`：页面结构。
- `quiz/index.html`：6 题测试页。
- `result/index.html`：免费基础结果页。
- `pay/index.html`：模拟解锁页，不接真实支付。
- `success/index.html`：完整住宿决策卡。
- `styles.css`：界面样式。
- `app.js`：6 题问卷、规则打分、结果存储、页面渲染。
- `data/boracay-hotels.js`：人工维护的酒店候选库。

第一版不用数据库，不需要登录，也不接地图。

测试结果存在浏览器 `localStorage`，只是本机演示用。

---

## 判断原则

每个答案只做简单加减分。

例如：

```text
用户选择：
- 躺平，看海，放空
- 怕吵，睡不好
- 白天热闹可以，晚上要安静
- 5-10 分钟可以接受
- 贵一点可以，但要明显更舒服
- 情侣 / 夫妻

则更偏向：
Station 1、Diniwid

同时降低：
Station 2 核心、Bulabog、Yapak 深处
```

这不是复杂算法。第一版只需要足够清楚、足够可改。

---

## 酒店候选

酒店候选不是 AI 编出来的。

它们来自静态候选库：

```text
data/boracay-hotels.js
```

候选来源记录在：

```text
data/hotel-sources.md
```

AI 或前端只能从候选库里挑，不能凭空发明酒店。

预订前仍然要重新检查：

- 近一年评价
- 价格
- 取消政策
- 噪音
- Wi-Fi
- 房间新旧
- 步行距离

## 暂不做

- 登录
- 收藏
- 地图
- 多轮对话
- 复杂搜索
- 自动爬虫
- 酒店实时价格
- 完整行程规划
- 真实支付接口
- 订单系统
- 用户系统

当前目标只有一个：

> 验证用户愿不愿意做完 6 题，并认为完整住宿决策卡值得解锁。
