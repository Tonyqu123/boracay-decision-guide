# Boracay Decision Guide

一个小而专注的 AI 旅行决策项目。

它不想成为新的旅游平台，也不想重做 Google Maps、携程、Klook、小红书。

它的目标是：

> 把分散在地图、OTA、内容社区、本地商家里的信息，整理成适合中文自由行用户的中立判断。

第一个垂直场景：

> 中文游客第一次去长滩岛，如何选择住哪里、怎么去、玩什么、是否潜水，以及如何避免被信息不对称消耗。

---

## 核心理念

### 用户不是缺信息，而是缺判断

旅游信息已经很多：

- Google Maps 有地点、评分、评论、照片。
- 携程 / Klook / Agoda 有商品、价格、订单、支付。
- 小红书 / 马蜂窝有体验、攻略、种草、踩坑。
- Facebook / Messenger / 本地商家有一手服务信息。

但用户真正痛苦的是：

- 信息分散。
- 信息不可比。
- 平台立场不中立。
- 评价缺少上下文。
- 不知道哪个选择适合自己。

所以这个项目不是做“更多攻略”，而是做：

> 旅行场景里的判断引擎。

---

## 产品原则

1. **不为流量，为用户省时间。** 目标不是让用户一直刷，而是尽快帮助用户完成判断，然后去休息、去旅行。
2. **不替代平台，而是帮助用户使用平台。** Google Maps、携程、Klook、小红书都是信息源，不是敌人。
3. **不做导购，做买方顾问。** 推荐要站在用户侧，而不是商家侧。
4. **不追求“最好”，追求“适合你”。** 旅游不是绝对排名问题，而是低后悔决策问题。
5. **必须表达不确定性。** 所有判断都要标注依据、风险、缺失信息和置信度。
